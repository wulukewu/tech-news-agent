## 設計原則

### 1. 資訊架構

**分組邏輯：**

- **主選單** - 核心功能，使用頻率高
  - 文章（瀏覽內容）
  - 閱讀清單（稍後閱讀）
  - 訂閱（管理來源）

- **更多功能** - 進階功能，使用頻率中等
  - 推薦（發現新內容）
  - 分析（查看統計）
  - 設定（個人化設定）
  - 系統狀態（監控系統）

### 2. 視覺設計

**分組標題樣式：**

```css
text-xs           /* 小字體 */
font-semibold     /* 粗體 */
text-muted-foreground  /* 灰色 */
uppercase         /* 大寫 */
tracking-wider    /* 字母間距 */
```

**導航項目樣式：**

```css
rounded-lg        /* 圓角 */
min-h-[48px]      /* 最小高度（觸控友善）*/
gap-3             /* 圖示與文字間距 */
transition-colors /* 平滑過渡 */
```

### 3. 互動設計

**狀態反饋：**

- **預設** - 透明背景
- **懸停** - `bg-accent` 背景
- **啟用** - `bg-primary` 背景 + 左側邊框
- **焦點** - 2px 環形邊框

**觸控目標：**

- 最小高度 48px（符合 WCAG 標準）
- 全寬度可點擊
- 適當的間距避免誤觸

## 測試場景

### 場景 1：新用戶首次使用

**原有設計：**

1. 點擊漢堡選單 → 看到 3 個選項
2. 疑惑：「其他功能在哪裡？」
3. 嘗試點擊頭像 → 發現更多功能
4. 需要記住功能分佈

**新設計：**

1. 點擊漢堡選單 → 看到所有 7 個選項
2. 清晰的分組標題幫助理解
3. 一次就能掌握所有功能
4. 不需要額外探索

### 場景 2：訪問次要功能

**原有設計：**

1. 想要查看分析
2. 點擊頭像 → 打開下拉選單
3. 找到「分析」選項
4. 點擊進入

**新設計：**

1. 想要查看分析
2. 點擊漢堡選單 → 打開抽屜
3. 在「更多功能」區域找到「分析」
4. 點擊進入

### 場景 3：快速切換功能

**原有設計：**

- 主要功能之間切換：快速（都在抽屜）
- 主要 ↔ 次要功能切換：慢（需要關閉一個打開另一個）

**新設計：**

- 所有功能之間切換：快速（都在抽屜）
- 一次打開，隨意切換
