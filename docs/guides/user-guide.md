## 快速開始

### 第一次使用

當你第一次執行任何指令時，系統會自動為你建立帳號。你不需要手動註冊！

### 基本流程

1. **訂閱 RSS 來源** - 使用 `/add_feed` 訂閱你感興趣的技術新聞來源
2. **查看文章** - 使用 `/news_now` 查看最新文章
3. **儲存到閱讀清單** - 點擊「⭐ 稍後閱讀」按鈕
4. **管理閱讀清單** - 使用 `/reading_list view` 查看和評分
5. **獲得推薦** - 使用 `/reading_list recommend` 獲得個人化推薦

## 訂閱管理

### 訂閱新的 RSS 來源

使用 `/add_feed` 指令訂閱 RSS 來源：

```
/add_feed name:Hacker News url:https://news.ycombinator.com/rss category:Tech News
```

**參數說明：**

- `name`: 來源名稱（例如：Hacker News、TechCrunch）
- `url`: RSS/Atom feed 網址（必須是 http:// 或 https://）
- `category`: 分類（例如：AI、Web、Security）

**注意事項：**

- URL 必須是有效的 HTTP/HTTPS 網址
- 如果你已經訂閱過該來源，系統會提示你
- 建議使用一致的分類名稱，方便之後篩選

### 查看訂閱清單

使用 `/list_feeds` 查看你訂閱的所有來源：

```
/list_feeds
```

系統會顯示：

- 來源名稱
- RSS 網址
- 分類
- 訂閱時間（最新的在前面）

**特色：**

- 按分類分組顯示
- 只有你能看到（ephemeral 回應）
- 顯示總訂閱數

### 取消訂閱

使用 `/unsubscribe_feed` 取消訂閱：

```
/unsubscribe_feed feed_identifier:Hacker News
```

**參數說明：**

- `feed_identifier`: 來源名稱或 ID

**注意事項：**

- 取消訂閱不會刪除該來源的文章
- 其他使用者仍可以訂閱該來源
- 如果找不到該來源，系統會提示你使用 `/list_feeds` 查看

## 查看文章

### 使用 /news_now

`/news_now` 指令會顯示你訂閱的最新技術文章：

```
/news_now
```

**運作方式：**

1. 查詢你訂閱的來源
2. 篩選最近 7 天的文章
3. 只顯示已經過 AI 分析的文章
4. 按技術深度（tinkering_index）排序
5. 限制顯示前 20 篇文章

**顯示格式：**

- 按分類分組
- 顯示火焰圖示（🔥）表示技術深度
- 包含文章標題和連結

### 互動元件

#### 📋 分類篩選選單

- 點擊選單可以按分類篩選文章
- 選項包括「顯示全部」和最多 24 個分類
- 篩選結果只有你能看到

#### 📖 深度分析按鈕

- 最多顯示 5 個按鈕（前 5 篇文章）
- 點擊後會生成詳細的技術分析
- 分析內容包括：
  - 核心技術概念
  - 應用場景
  - 潛在風險
  - 建議的下一步

#### ⭐ 稍後閱讀按鈕

- 最多顯示 10 個按鈕（前 10 篇文章）
- 點擊後會將文章加入你的閱讀清單
- 按鈕會在點擊後禁用，避免重複加入

### 注意事項

- 文章由背景排程器抓取（每週五 17:00）
- 如果你沒有訂閱任何來源，系統會提示你先訂閱
- 如果沒有新文章，系統會提示你等待下次抓取

## 閱讀清單

### 查看閱讀清單

使用 `/reading_list view` 查看你的閱讀清單：

```
/reading_list view
```

**功能：**

- **分頁瀏覽**：每頁顯示 5 篇文章
- **上一頁/下一頁**：使用按鈕導航
- **標記已讀**：點擊 ✅ 按鈕標記文章為已讀
- **評分**：使用下拉選單評分（1-5 星）

**顯示資訊：**

- 文章標題和連結
- 分類
- 目前評分（或「未評分」）

### 標記已讀

點擊 ✅ 按鈕可以將文章標記為已讀：

- 按鈕會在點擊後禁用
- 系統會顯示確認訊息
- 文章狀態會更新為「Read」

### 評分文章

使用下拉選單為文章評分：

- **⭐** (1 星) - 不推薦
- **⭐⭐** (2 星) - 普通
- **⭐⭐⭐** (3 星) - 還可以
- **⭐⭐⭐⭐** (4 星) - 推薦
- **⭐⭐⭐⭐⭐** (5 星) - 強烈推薦

**評分的用途：**

- 建立你的閱讀偏好檔案
- 用於生成個人化推薦
- 追蹤你喜歡的文章類型

## 個人化推薦

### 獲得推薦

使用 `/reading_list recommend` 獲得個人化推薦：

```
/reading_list recommend
```

**運作方式：**

1. 查詢你評分 4 星或 5 星的文章
2. 分析文章標題和分類
3. 使用 AI 生成個人化推薦摘要
4. 建議你接下來應該閱讀什麼

**注意事項：**

- 你需要至少有一篇 4 星或 5 星的文章
- 推薦內容使用繁體中文
- 推薦會根據你的評分歷史調整

## 常見問題

### Q: 為什麼 /news_now 沒有顯示文章？

**A:** 可能的原因：

1. 你還沒有訂閱任何 RSS 來源 → 使用 `/add_feed` 訂閱
2. 背景排程器還沒有執行 → 等待每週五 17:00
3. 訂閱的來源沒有新文章 → 嘗試訂閱更多來源

### Q: 按鈕點擊後沒有反應？

**A:** 可能的原因：

1. 按鈕已經被點擊過（會顯示為禁用狀態）
2. Bot 重啟後舊訊息的按鈕可能失效 → 重新執行指令
3. 網路延遲 → 稍等幾秒再試

### Q: 如何刪除閱讀清單中的文章？

**A:** 目前系統支援：

- 標記為已讀（使用 ✅ 按鈕）
- 標記為封存（需要手動更新資料庫）

未來版本會加入刪除功能。

### Q: 可以訂閱多少個 RSS 來源？

**A:** 沒有限制！你可以訂閱任意數量的 RSS 來源。

### Q: 評分會影響其他使用者嗎？

**A:** 不會！每個使用者的評分和閱讀清單都是獨立的。你的評分只會影響你自己的推薦。

### Q: 如何更改訂閱的分類？

**A:** 目前需要：

1. 取消訂閱該來源（`/unsubscribe_feed`）
2. 重新訂閱並使用新的分類（`/add_feed`）

### Q: 深度分析按鈕會消耗什麼？

**A:** 每次點擊深度分析按鈕會：

- 呼叫 Groq API（使用 Llama 3.3 70B 模型）
- 生成最多 600 個 token 的分析
- 消耗 API 配額

建議只對真正感興趣的文章使用深度分析。

### Q: 閱讀清單有數量限制嗎？

**A:** 沒有限制！你可以加入任意數量的文章到閱讀清單。

### Q: 如何查看已讀的文章？

**A:** 目前 `/reading_list view` 只顯示未讀文章。查看已讀文章需要直接查詢資料庫。

未來版本會加入狀態篩選功能。

## 提示與最佳實踐

### 訂閱管理

1. **選擇高品質來源**：訂閱技術深度高的 RSS 來源
2. **使用一致的分類**：在不同來源使用相同的分類名稱，方便篩選
3. **定期檢查訂閱**：使用 `/list_feeds` 檢查並清理不再需要的訂閱

### 閱讀習慣

1. **定期評分**：養成評分習慣，建立你的偏好檔案
2. **善用深度分析**：只對真正感興趣的文章使用深度分析
3. **保持清單整潔**：定期標記已讀文章

### 個人化推薦

1. **評分要誠實**：根據真實感受評分，不要隨意給高分
2. **評分要多樣**：評分不同類型的文章，讓推薦更準確
3. **定期獲取推薦**：每週獲取一次推薦，了解你的閱讀趨勢
