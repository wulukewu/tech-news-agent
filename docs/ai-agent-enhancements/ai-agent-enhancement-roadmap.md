# AI Agent 增強方案執行路線圖

## 📋 執行順序總覽

基於價值優先級、技術複雜度和依賴關係，建議按以下順序執行：

### ✅ Phase 0.5: Agent 存在感補強 (已插入，優先於 Phase 2)

> **背景**: Phase 1 完成後，用戶反映感受不到 agent 帶來的主要幫助。根本原因是現有功能都是「被動回應」，缺少 agent 主動行動的閉環。

#### 主動推薦閉環 ⭐⭐⭐⭐⭐

- **Spec 路徑**: `.kiro/specs/proactive-recommendation-loop/`
- **預估時間**: 2-3 天
- **優先級**: **最高（插入 Phase 2 之前）**
- **核心價值**: 讓用戶第一次感受到「這不是普通 chatbot，它會主動找我」
- **技術重點**:
  - fetch 完文章後自動分析用戶評分歷史
  - 主動發 Discord DM：「根據你的偏好，今天有 N 篇值得看，原因是 X」
  - 用戶可在 DM 裡直接回饋「這類不要推了」，agent 即時更新偏好
- **為什麼現在做**:
  - proactive-learning-agent 的基礎設施已完成（行為分析、偏好模型、權重調整）
  - 缺的只是「主動送出通知」這個最後一哩路
  - 沒有這個閉環，Phase 2 的學習路徑和知識圖譜都是空中樓閣（用戶不會去評分，數據不會累積）

---

### 🚀 Phase 1: 核心對話功能 (2-3 週)

#### 1. 智能問答 Agent ⭐⭐⭐⭐⭐

- **Spec 路徑**: `.kiro/specs/intelligent-qa-agent/`
- **預估時間**: 3-4 天
- **優先級**: 最高
- **核心價值**: 從「推送工具」升級為「智能助理」
- **技術重點**: RAG 架構、pgvector、LangChain
- **為什麼優先**: 最直接的價值提升，用戶可以主動探索內容

#### 1.5. 聊天記錄持久化與跨平台同步 ⭐⭐⭐⭐⭐⭐

- **設計文件**: `docs/chat-persistence-design.md`
- **預估時間**: 4-5 天
- **優先級**: **極高** (比其他 Phase 1 功能更重要)
- **核心價值**:
  - 🔄 Web 和 Discord 無縫對話同步
  - 📚 完整對話歷史管理和搜尋
  - 🎯 真正的連續對話體驗
- **技術重點**:
  - 跨平台用戶身份綁定
  - 對話持久化和同步機制
  - Discord Bot 整合
  - 智能對話管理功能
- **為什麼極高優先級**:
  - 這是所有 AI Agent 功能的**基礎設施**
  - 直接影響用戶體驗的連續性
  - 沒有這個功能，用戶無法真正「接續對話」
  - 比單純的問答功能更重要，因為它解決了**用戶體驗的根本問題**

#### 2. 每週洞察摘要 Agent ⭐⭐⭐⭐

- **Spec 路徑**: `.kiro/specs/weekly-insights-summary-agent/`
- **預估時間**: 2-3 天
- **優先級**: 高
- **核心價值**: 展示 AI 分析能力，增強現有通知功能
- **技術重點**: 跨文章主題分析、趨勢檢測
- **為什麼第二**: 增強現有功能，相對容易實作，快速見效

#### 3. 主動學習 Agent ⭐⭐⭐⭐

- **Spec 路徑**: `.kiro/specs/proactive-learning-agent/`
- **預估時間**: 3-4 天
- **優先級**: 高
- **核心價值**: 改善推薦準確度，增加用戶參與度
- **技術重點**: 行為分析、對話管理、權重調整
- **為什麼第三**: 需要前面兩個功能的數據基礎，但能顯著改善推薦系統

### 🎯 Phase 2: 進階學習功能 (3-4 週)

#### 4. 學習路徑規劃 Agent ⭐⭐⭐⭐

- **Spec 路徑**: `.kiro/specs/learning-path-planning-agent/`
- **預估時間**: 4-5 天
- **優先級**: 中高
- **核心價值**: 提供結構化學習體驗
- **技術重點**: 技能樹建模、路徑生成、進度追蹤
- **為什麼第四**: 依賴智能問答和主動學習的數據，提供更高級的學習管理

#### 5. 智能提醒 Agent ⭐⭐⭐

- **Spec 路徑**: `.kiro/specs/intelligent-reminder-agent/`
- **預估時間**: 3-4 天
- **優先級**: 中
- **核心價值**: 提高內容消費效率，減少無效打擾
- **技術重點**: 文章關聯分析、時機判斷、個人化提醒
- **為什麼第五**: 需要用戶行為數據積累，優化現有推送體驗

#### 6. 知識圖譜 Agent ⭐⭐⭐⭐⭐

- **Spec 路徑**: `.kiro/specs/knowledge-graph-agent/`
- **預估時間**: 5-6 天
- **優先級**: 中高
- **核心價值**: 提供清晰的學習路徑視覺化
- **技術重點**: LLM 依賴提取、圖形資料庫、D3.js 視覺化
- **為什麼最後**: 技術複雜度最高，需要前面所有功能的數據支撐

## 🎯 執行策略

### 快速見效策略 (推薦)

如果要在 **1-2 週內**看到顯著改善：

1. **智能問答 Agent** (3-4 天) - 基礎問答功能
2. **聊天記錄持久化與跨平台同步** (4-5 天) - **核心基礎設施**
3. **每週洞察摘要 Agent** (2-3 天) - 增強分析能力

**重要**: 聊天記錄持久化功能雖然需要更多時間，但它是**所有後續功能的基礎**，沒有它就無法提供真正連續的對話體驗。這個投資是必要的。

### 完整實作策略

按照 1-6 的順序依次完成，每個功能都會為後續功能提供數據基礎和用戶體驗改善。

## 📊 依賴關係圖

```
智能問答 Agent (基礎 RAG 架構)
    ↓
聊天記錄持久化與跨平台同步 (核心基礎設施) ⭐ 關鍵基礎
    ↓
每週洞察摘要 Agent (文章分析能力)
    ↓
主動學習 Agent (用戶行為分析)
    ↓
學習路徑規劃 Agent (結構化學習)
    ↓
智能提醒 Agent (個人化推送)
    ↓
知識圖譜 Agent (視覺化整合)
```

**重要說明**: 聊天記錄持久化功能是所有後續 AI Agent 功能的基礎設施。沒有它：

- 用戶無法在不同平台間接續對話
- 無法建立長期的學習歷史
- AI Agent 無法提供真正個人化的體驗
- 對話體驗會支離破碎
