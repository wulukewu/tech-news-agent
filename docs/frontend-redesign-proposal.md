## 專案概述

**Tech News Agent** 是一個技術資訊訂閱與管理平台，讓使用者可以：

- 訂閱 RSS 技術資訊來源
- 瀏覽和管理文章
- 建立個人閱讀清單
- 接收個人化推薦
- 查看系統狀態和分析數據

**技術棧：**

- Frontend: Next.js 14, React 18, TypeScript, Tailwind CSS, shadcn/ui
- Backend: FastAPI (Python)
- 認證: Discord OAuth2

## 目前問題分析

### 🔴 嚴重問題

#### 1. 導航結構混亂

**問題描述：**

- 從截圖可以看到，左側有一個側邊欄，但側邊欄裡面又有另一個側邊欄
- 導航項目過多且沒有明確的層級關係
- 使用者需要點擊多次才能到達目標頁面

**影響：**

- 使用者體驗差，容易迷失方向
- 增加認知負擔
- 降低操作效率

#### 2. 資訊架構不清晰

**問題描述：**

- 8 個主要導航項目全部平鋪在同一層級
- 沒有區分核心功能和輔助功能
- 功能分組不明確

**影響：**

- 使用者難以快速找到需要的功能
- 介面看起來雜亂無章

#### 3. 行動版體驗不佳

**問題描述：**

- 雖然有響應式設計，但行動版的導航抽屜包含過多項目
- 觸控目標可能不夠大
- 資訊密度過高

**影響：**

- 行動裝置上操作困難
- 容易誤觸

### 🟡 中等問題

#### 4. 視覺層級不明確

**問題描述：**

- 主要內容區域和次要功能區域沒有明顯區分
- 文章卡片的視覺權重不夠
- 缺乏視覺焦點

#### 5. 功能重複或冗餘

**問題描述：**

- Dashboard 和 Articles 頁面功能重疊
- 某些功能可以合併或簡化

#### 6. 缺乏使用者引導

**問題描述：**

- 新使用者不知道從哪裡開始
- 沒有 Onboarding 流程
- 功能說明不足

### 🟢 輕微問題

#### 7. 主題切換位置不一致

- 桌面版和行動版的主題切換按鈕位置不同

#### 8. 載入狀態不夠友善

- 某些頁面的載入狀態過於簡單

## 新版設計架構

### 設計原則

1. **簡化優先 (Simplicity First)**
   - 減少導航層級
   - 合併相似功能
   - 清晰的資訊架構

2. **內容為王 (Content First)**
   - 文章內容是核心
   - 減少干擾元素
   - 提升閱讀體驗

3. **行動優先 (Mobile First)**
   - 優先考慮行動裝置體驗
   - 響應式設計
   - 觸控友善

4. **一致性 (Consistency)**
   - 統一的視覺語言
   - 一致的交互模式
   - 可預測的行為

### 核心改進方向

#### 1. 扁平化導航結構

- **移除巢狀側邊欄**
- 採用單一導航系統
- 減少點擊次數

#### 2. 功能分組與優先級

將功能分為三個層級：

**核心功能 (主導航)**

- 📰 文章瀏覽 (Articles)
- 📚 閱讀清單 (Reading List)
- ⚙️ 訂閱管理 (Subscriptions)

**次要功能 (次導航或標籤頁)**

- 💡 推薦 (Recommendations) - 整合到文章瀏覽
- 📊 分析 (Analytics) - 整合到個人中心
- 🔔 通知設定 (Notifications) - 整合到設定

**輔助功能 (下拉選單或設定頁)**

- 🖥️ 系統狀態 (System Status)
- ⚙️ 設定 (Settings)
- 👤 個人資料 (Profile)

#### 3. 新的頁面架構

```
┌─────────────────────────────────────────┐
│  頂部導航列 (固定)                        │
│  Logo | 搜尋 | 主導航 | 使用者選單        │
├─────────────────────────────────────────┤
│                                         │
│  主要內容區域                            │
│  (根據不同頁面顯示不同內容)               │
│                                         │
│                                         │
└─────────────────────────────────────────┘
```

## 功能與交互設計

### 1. 搜尋功能

#### 桌面版

- 搜尋列固定在頂部導航列
- 即時搜尋 (輸入時即時顯示結果)
- 支援搜尋文章標題、內容、來源、分類
- 搜尋結果以下拉清單顯示，點擊進入文章詳情

#### 行動版

- 搜尋圖示在頂部導航列
- 點擊後展開全螢幕搜尋介面
- 搜尋歷史記錄
- 熱門搜尋建議

### 2. 文章卡片交互

#### 桌面版

- **Hover 效果**: 卡片陰影加深、輕微上移
- **點擊標題**: 開啟文章詳情頁面
- **點擊來源**: 篩選該來源的所有文章
- **點擊分類**: 篩選該分類的所有文章
- **操作按鈕**:
  - 稍後閱讀: 加入閱讀清單
  - 標記已讀: 標記為已讀狀態
  - 分享: 複製連結或分享到社群媒體

#### 行動版

- **長按**: 顯示快速操作選單
- **左滑**: 稍後閱讀
- **右滑**: 標記已讀
- **點擊**: 開啟文章詳情

### 3. 篩選和排序

#### 分類篩選

- 多選分類標籤
- 選中的標籤高亮顯示
- 可以清除所有篩選

#### 排序選項

- 最新 (預設)
- 最熱門 (根據互動數)
- Tinkering Index 高到低
- Tinkering Index 低到高

#### 檢視模式

- **卡片檢視** (預設): 大卡片，顯示完整資訊
- **列表檢視**: 緊湊列表，適合快速瀏覽
- **緊湊檢視**: 最小化資訊，顯示更多文章

### 4. 無限滾動

- 滾動到底部自動載入更多文章
- 顯示載入指示器
- 載入失敗時顯示重試按鈕
- 沒有更多文章時顯示提示

### 5. 通知系統

#### 通知類型

- 新文章通知
- 推薦文章通知
- 系統通知

#### 通知設定

- 全域開關
- 每個 Feed 的通知開關
- Tinkering Index 閾值設定 (只通知高於某個深度的文章)
- 通知頻率設定 (即時、每小時、每天)
- 安靜時段設定

### 6. 主題切換

- 明亮模式
- 深色模式
- 跟隨系統設定
- 平滑過渡動畫

### 7. 離線支援 (PWA)

- 快取已載入的文章
- 離線時顯示已快取的內容
- 離線時禁用需要網路的功能
- 恢復網路時自動同步

## 技術實作方向

### 1. 導航系統重構

#### 目標

- 移除巢狀側邊欄
- 統一桌面版和行動版的導航邏輯
- 減少導航項目數量

#### 實作步驟

1. 更新 `Navigation.tsx` 組件
   - 簡化導航項目為 3 個核心功能
   - 將次要功能移到使用者選單
   - 優化行動版抽屜的佈局

2. 創建 `UserMenu.tsx` 組件
   - 下拉選單包含次要功能
   - 個人資料、統計、設定、系統狀態
   - 主題切換和登出

3. 更新路由結構
   - 移除不必要的巢狀路由
   - 簡化 URL 結構

### 2. 頁面整合

#### 整合 Dashboard 和 Articles

## 路徑架構設計

### 目前路徑問題

1. **首頁即登入頁**: `/` 直接是登入頁面，沒有獨立的首頁
2. **強制跳轉**: 已登入使用者訪問 `/` 會被強制跳轉到 `/dashboard`
3. **路徑語意不清**: `/dashboard` 這個名稱不夠直觀
4. **巢狀路由混亂**: `(dashboard)` 群組路由造成路徑不一致

### 新版路徑架構

#### 公開路徑 (未登入可訪問)

```

/login                     → 登入頁面

```

#### 應用路徑 (需要登入)

export default function AppLayout({ children }) {
  const { isAuthenticated, loading } = useAuth();
  const router = useRouter();
  const pathname = usePathname();

  useEffect(() => {
    if (!loading && !isAuthenticated) {
      router.push(`/login?redirect=${pathname}`);
    }
  }, [isAuthenticated, loading]);

  if (loading) return <LoadingScreen />;

  );
}
