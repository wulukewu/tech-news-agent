## 可能的問題

### 1. 時間設定問題

如果你設定的是：

- **早上 6:00（台北時間）** → 應該是 UTC 22:00（前一天）
- **早上 10:00（台北時間）** → 應該是 UTC 02:00 ✅ 符合當前排程

**請確認**：你在前端設定的通知時間是幾點？

### 2. 時區設定問題

請確認：

- 你的時區設定是 `Asia/Taipei` 嗎？
- 如果設定的是 `UTC`，那時間會不一樣

### 3. 時間已過，排程到明天

如果你設定的時間已經過了（例如現在是 23:35，你設定的是 22:00），系統會自動排程到明天的同一時間。

## 調試步驟

### 步驟 1: 檢查當前設定

```bash
curl -s http://localhost:8000/api/debug/scheduler/jobs | python3 -m json.tool
```

找到你的 job，查看 `next_run_time`。

### 步驟 2: 檢查資料庫中的設定

在前端查看 Settings > Notifications：

- 通知時間：**:**
- 時區：**\_\_\_\_**
- 頻率：**\_\_\_\_**

### 步驟 3: 手動觸發測試

1. 修改通知時間為「當前時間 + 2 分鐘」
2. 點擊「重新排程」按鈕
3. 等待 2 分鐘
4. 檢查是否收到通知

### 步驟 4: 監控日誌

```bash
# 實時監控排程執行
docker logs -f tech-news-agent-backend-dev | grep -i "sending scheduled notification"
```

當時間到達時，應該會看到：

```
Sending scheduled notification to user
```

## 時區轉換參考

| 台北時間 | UTC 時間       | 說明       |
| -------- | -------------- | ---------- |
| 00:00    | 16:00 (前一天) | 午夜       |
| 06:00    | 22:00 (前一天) | 早上 6 點  |
| 10:00    | 02:00          | 早上 10 點 |

## 預期行為

### Daily（每日）

- 每天在設定的時間發送一次
- 如果當天時間已過，排程到明天

### Weekly（每週）

- 每週五在設定的時間發送一次
- 如果今天是週五且時間已過，排程到下週五

### Monthly（每月）

- 每月 1 號在設定的時間發送一次
- 如果今天是 1 號且時間已過，排程到下個月 1 號

## 常見問題

### Q: 為什麼我設定早上 6 點，但 job 顯示 UTC 02:00？

A: 因為 UTC 02:00 = 台北時間 10:00，不是 6 點。請檢查你的設定。

### Q: 我設定的時間到了，但沒收到通知

A: 可能原因：

1. 時間設定錯誤（檢查時區）
2. Bot 未就緒（檢查 Bot 狀態）
3. DM 未啟用（檢查通知管道設定）
4. 在勿擾時段內（檢查勿擾設定）

### Q: 如何確認排程是否會執行？

A:

1. 設定時間為 2 分鐘後
2. 監控日誌：`docker logs -f tech-news-agent-backend-dev`
3. 等待並觀察
